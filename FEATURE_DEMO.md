## 🔐 云端数据同步演示

### 第 1 步: 登录账号

1. 打开应用
2. 点击底部导航栏的"设置"
3. 在"账号同步"部分，点击"登录账号"
4. 输入邮箱和密码
5. 点击"登录"

**示例账号**:
- 邮箱: `test@example.com`
- 密码: `password123`

### 第 2 步: 启用云端同步

1. 登录成功后，会显示"已登录: test@example.com"
2. 点击"🔄"按钮启用云端同步
3. 同步状态会显示"🔄 同步中..."
4. 同步完成后显示"✓ 已同步"

### 第 3 步: 测试数据同步

1. 进入首页
2. 输入体重和身高
3. 记录健康状态
4. 点击"保存本周记录"
5. 观察同步状态指示器

**同步状态说明**:
- 📱 离线 - 未登录或未启用同步
- 🔄 同步中 - 正在上传/下载数据
- ✓ 已同步 - 数据同步成功
- ✗ 同步失败 - 网络错误或同步失败

### 第 4 步: 验证多设备同步

1. 在设备 A 上登录并输入数据
2. 保存数据（自动同步）
3. 在设备 B 上登录同一账号
4. 进入首页，观察数据是否同步

**预期效果**:
- ✓ 设备 B 显示设备 A 的数据
- ✓ 数据完全一致
- ✓ 支持跨设备访问

## 🔄 自动同步机制

### 同步触发条件

1. **手动保存**
   - 点击"保存本周记录"按钮
   - 立即同步到云端

2. **自动同步**
   - 每 5 分钟自动同步一次
   - 后台自动运行，无需用户操作

3. **登录时同步**
   - 登录账号时自动从云端下载数据
   - 确保数据最新

### 同步流程

```
用户操作
   ↓
保存数据到 LocalStorage
   ↓
检查是否启用云端同步
   ↓
是 → 上传到 JSONBin.io
   ↓
显示同步状态
   ↓
完成
```

## 🐛 常见问题

### Q: 同步失败怎么办？
A: 
1. 检查网络连接
2. 确认邮箱和密码正确
3. 刷新页面重试
4. 检查浏览器控制台错误信息

### Q: 数据会丢失吗？
A:
- 本地数据存储在 LocalStorage，清除浏览器缓存会丢失
- 云端数据存储在 JSONBin.io，不会丢失
- 建议启用云端同步作为备份

### Q: 支持多个账号吗？
A:
- 目前只支持一个账号登录
- 切换账号需要先登出再登录

### Q: 如何导出数据？
A:
- 目前不支持直接导出
- 可以通过浏览器开发者工具导出 LocalStorage 数据
- 后续版本会添加导出功能
